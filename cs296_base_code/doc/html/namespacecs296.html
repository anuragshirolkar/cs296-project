struct  settings_t
 This class deals with Simulation settings especially those of the debug_draw and m_world
some of which can be controlled in the GUI.It ensures that overlappings don't occur,drawing flags are set etc.

A brief description of the members:

hz=60.0 fps is basically used for assigning the timeskip

The velocity and step iterations are only relevant to collision resolution. < br> They help in preventing any kind of overlap.

draw_shapes,draw_joints,draw_AABBs,draw_pairs and draw_COMs are draw flags for m_debug_draw

draw_contact* and draw_friction* are for simulation of contact forces,friction
and other other parameters as ther name suggests.

enable_* variables are meant to to start cetain properties like warm_starting,continuity,substepping<br>

The namespace protects the global variables and other names from clashes in scope. Read about the use of named and unnamed namespaces in C++ Figure out where all the datatypes used below are defined

This file defines all the callback functions our code needs. Why are they not members of a class? Can you make them into a class?

Callback functions are called when a event occurs due to the press of a key, mouse button, or a GUI element like a menu item is clicked or scroller is dragged.
